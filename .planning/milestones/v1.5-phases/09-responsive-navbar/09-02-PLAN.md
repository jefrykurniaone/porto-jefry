<objective>
Integrate the Plan 01 `MobileDrawer` into the Navbar, fix the root-cause overflow bug, and add the horizontal-scroll fallback for the inline nav.

Root cause (from the locked design): `<sgds-mainnav expand="always">` forces the desktop inline layout at every width, so all 7 links render inline, overflow below md, and get clipped — and the hamburger + toggles in the `end` slot get pushed off-screen. This plan stops forcing the always-expanded layout, collapses the phone bar to brand + hamburger only, opens the new drawer from the hamburger, and makes the md+ inline nav scroll horizontally when items do not all fit.

Purpose: This is the integration wave — it consumes the self-contained drawer/hooks/i18n from Plan 01 and delivers the user-visible responsive behavior across phone, tablet, and desktop.

Output: Updated `Navbar.tsx` and extended `Navbar.test.tsx`; an optional new i18n label for the scroll affordance if one is exposed.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client -> URL hash | `scrollTo` writes `history.pushState(null, '', '#<id>')` where `<id>` comes only from the fixed `NAV_KEYS` allowlist — no user-controlled value reaches pushState. |
| client -> DOM scroll | The drawer's scroll lock (owned by MobileDrawer via useScrollLock) toggles document overflow; must always restore on close. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-04 | Tampering | scrollTo / pushState hash | accept | Hash is derived from the hardcoded `NAV_KEYS` allowlist via `getElementById(id)?.scrollIntoView()`; no user input, so no hash-injection or open-redirect risk. |
| T-09-05 | Denial of Service | drawer scroll lock | mitigate | Scroll lock lives in `useScrollLock` (Plan 01) which restores the prior overflow on close/unmount; Navbar guarantees `onClose` is reachable via Esc, close button, and backdrop, so the lock cannot strand the page. |
| T-09-06 | Elevation of Privilege | focus trap escaping | mitigate | A single focus-trap owner (MobileDrawer's `useFocusTrap`) wraps Tab within the panel and restores focus to the hamburger on close; Navbar removes its duplicate trap to prevent inconsistent focus state. |
| T-09-SC | Tampering | npm/pip/cargo installs | mitigate | No new dependencies installed; reuses existing SGDS, next-intl, React, and the Plan 01 hooks. No package-legitimacy checkpoint required. |

No high-severity threats: static labels, no user input into hrefs/pushState, no new network calls, no new dependencies.
</threat_model>
