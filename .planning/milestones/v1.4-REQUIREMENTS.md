# Milestone v1.4 Requirements — Polish & International Content

**Opened:** 2026-06-20
**Follows:** v1.3 (Improvement & Hardening). Numbering aligned to git tags (v1.1, v1.2 shipped; v1.3 = the improvement+SGDS milestone).
**Core Value:** A fast, accessible, bilingual portfolio that accurately represents Jefry's work and makes it easy for recruiters and collaborators — including international employers — to download his CV and reach him.

This is a point-in-time snapshot of the v1.4 milestone scope. The live, evolving copy is `.planning/REQUIREMENTS.md`.

## Why this milestone

Owner feedback on the shipped SGDS monochrome UI:
- Muted gray text is hard to read (low contrast).
- The three hero CTA buttons are unbalanced and "View My Work" wraps.
- The theme/language toggle buttons feel oversized.
- The GitHub link belongs in About, not Contact.
- The content reads like generic resume-speak and should appeal to international employers.

## Requirements

### Look & Feel

- **UI-05**: All `sgds:text-muted` text meets WCAG AA contrast (≥4.5:1) in both light and dark themes. Root cause: SGDS token `--sgds-color-muted` resolves to `#c6c6c6` (day) / `#3b3b3b` (night) ≈ 1.6:1. Fix via a single override in `globals.css`.
- **UI-06**: Hero shows three equal-sized CTA buttons; "View My Work" never wraps; buttons stack full-width below 512px. Root cause: anchors have a fixed `width: 9.25rem` + no `white-space: nowrap`, and the `sgds-button` CV button is sized by a different rule (44px vs 48px, 168px vs 148px).
- **UI-07**: Theme toggle is a sun/moon sliding switch (`sgds-switch`, already globally registered); language toggle is a compact segmented EN|ID pill. Both compact and in the navbar.

### Information Architecture

- **IA-01**: GitHub link rendered in the About section and removed from the Contact section. URL stays defined once in `src/data/contact.ts`; label key moves to `about.*` (EN + ID parity).

### Content

- **CONTENT-01**: Core prose (hero subtitle/title, about summary, all experience bullets, contact, certification) rewritten in a natural, non-buzzword voice via a humanizer skill (installed via `find-skills`); includes a remote/international availability signal; no fabricated metrics (qualitative impact + `[your number]` placeholders); EN/ID key parity preserved.
- **CONTENT-02**: All 14 projects have bilingual descriptions sourced from i18n (`projects.items.<id>.description`); `Projects.tsx` reads them via next-intl with a missing-key guard.
- **CONTENT-03**: Education majors/degrees use idiomatic English — "Diploma in Informatics Management" (D3 Manajemen Informatika) and "Bachelor's in Information Systems" (S1 Sistem Informasi).
