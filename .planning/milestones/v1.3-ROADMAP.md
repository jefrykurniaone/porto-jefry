## Milestone Summary

### Key Accomplishments

1. **Full SGDS migration** — every UI component moved to `@govtechsg/sgds-web-component`; Tailwind upgraded v3→v4; `next-themes` removed in favor of SGDS `.sgds-night-theme` theming; flattened to a minimalist monochrome look.
2. **Security hardening** — rate-limit identity from trusted `ipAddress()` only (spoofed `x-forwarded-for` neutralized); CSP tightened (Google Fonts removed, theme-init hash recomputed and verified against the live script); distributed rate limiting closed as documented accepted-risk.
3. **Type safety & code quality** — eliminated the `useMessages()` double-cast via an exported `ExperienceMessages` interface; automated `LAST_MODIFIED_DATE` at build time; activated `eslint-plugin-sonarjs` error-level gates (max-lines, max-lines-per-function, no-nested-template-literals); extracted shared `TECH_*` constants.
4. **UX polish** — inline CV-download error feedback; branded locale-aware 404 and error pages; ThemeToggle CLS fixed.
5. **Quick correctness fixes** — button types, passive scroll listener, regex word-boundary anchors, consistent hash encoding.

### Key Decisions

- Keep SGDS but override only `--sgds-product-primary-*` to neutral grays for a monochrome look (#27).
- `next-themes` → SGDS theming via `.sgds-night-theme` class + FOUC-prevention inline script (hash-authorized in CSP).
- In-memory rate limiter retained as accepted-risk (per-instance Map adequate at portfolio scale; CV PDF module-cached).
- `t.raw('items') as ExperienceMessages['items']` for typed access to structured next-intl subtrees (single assertion replaces double cast).
- Private in-file sub-component extraction is the established pattern for the 40-line function limit.

### Technical Debt / Deferred

- UK English writing standards audit — deferred (future consideration).
- Masthead SGDS component — evaluated, not adopted (doesn't fit a personal portfolio).
