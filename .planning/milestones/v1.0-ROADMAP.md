## Milestone Summary

### Key Accomplishments

1. **IP source hardening** — Replaced `x-forwarded-for`-only extraction (which returned HTTP 400 on local dev) with `x-real-ip → x-forwarded-for[0] → 127.0.0.1` priority chain
2. **Bounded rate-limit memory** — Added lazy `evictExpiredEntries()` using `Map.prototype.forEach`; Map is now bounded to IPs active within the last 60s window
3. **RFC-compliant Retry-After** — Replaced hardcoded `'60'` with `Math.max(1, Math.ceil((windowStart + RATE_LIMIT_WINDOW_MS - Date.now()) / 1000))`
4. **URL-safe CV download** — Replaced template-literal URL with `new URLSearchParams({ locale })` in Hero.tsx

### Key Decisions

| Decision | Rationale | Outcome |
|----------|-----------|---------|
| `Map.forEach` over `for...of` for eviction | `tsconfig.json` has no explicit `target` (defaults ES3/ES5); `for...of` on Map triggers TS2802 without `downlevelIteration` | ✓ Good — identical semantics, zero config changes |
| `x-real-ip` as primary IP source | Vercel sets this header before route handler — non-spoofable verified client IP | ✓ Good |
| `Math.max(1, …)` guard on Retry-After | Sub-millisecond race: `Date.now()` may advance between `checkRateLimit` and response block | ✓ Good — guarantees valid RFC 9110 positive integer |
| Ship v1.0 with only Phase 1 complete | Phase 1 addresses the highest-severity production risks; Phases 2–5 are quality/debt improvements | ⚠️ Revisit — 15 requirements deferred |
