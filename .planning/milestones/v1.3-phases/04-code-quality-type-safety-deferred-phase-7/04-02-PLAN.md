<objective>
Automate `LAST_MODIFIED_DATE` so it reflects the actual last git commit date at build time
instead of being a manually-maintained string (QUAL-01). Per D-01/D-01a, derive a single
repo-wide date via `git log -1 --format=%cs` (YYYY-MM-DD) at BUILD time (not request/edge),
write it into `src/utils/constants.ts` via a small codegen script wired to npm's `prebuild`
hook, and include a safe fallback (current date) so the build never breaks if git is
unavailable. The export shape stays a `string` in `YYYY-MM-DD` form so `src/app/sitemap.ts`
needs no signature change.

Purpose: Make the sitemap `lastmod` truthful (changes only when content/code changes) and
remove a manual constant from the maintenance burden.
Output: `scripts/gen-build-meta.mjs`, an updated `src/utils/constants.ts`, and a `prebuild`
script in `package.json`.
</objective>

| T-04-04 | Denial of Service (build break) | missing/failed git at build | mitigate | Wrap git call in try/catch with a `new Date()` fallback and stderr warning (no empty catch) so the build always produces a valid YYYY-MM-DD and never fails. |
| T-04-05 | Information Disclosure | generated constants.ts | accept | The only data written is a commit date (already public in the repo history) and the existing public BASE_URL. No secrets are emitted. |

No `high`-severity threats are introduced. The single build-time git invocation is fully
parameterised with a fixed argument array (no injection) and has a safe fallback.
