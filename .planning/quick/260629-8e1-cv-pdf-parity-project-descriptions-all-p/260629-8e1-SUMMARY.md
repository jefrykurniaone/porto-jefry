# Phase 260629-8e1 Plan 01: CV PDF Parity (Descriptions, All Projects, GitHub) Summary

Brought the downloaded PDF CV (`/api/generate-cv`) to parity with the website: all 14 projects now render with localized descriptions and full tech lists, project periods are localized via `translatePeriod`, and the header shows a GitHub link after LinkedIn.

## What Was Built

**Task 1 ‚Äî Types & styles foundation (additive)** ‚Äî commit `75a1585`
- Added `items: Record<string, { description: string }>` to `Messages.projects` (cv-types.ts), mirroring the `experience.items` precedent and the runtime shape passed in from en/id.json.
- Added a `projectDesc` style (fontSize 8.5, DARK, lineHeight 1.4, marginBottom 3) modeled on `certDesc`.
- Re-exported `CONTACT_GITHUB_URL as GITHUB_URL` / `CONTACT_GITHUB_DISPLAY as GITHUB_DISPLAY` from cv-styles.

**Task 2 ‚Äî All projects, descriptions, full tech, localized periods** ‚Äî commit `d6aa4db`
- Rewrote `CvProjects.tsx`: iterates the full `projects` array (removed the 6-project cap), resolves each project's description via `messages.projects.items[proj.id]?.description`, renders it conditionally between meta and tech rows, maps the full `proj.tech` (removed the `+N` truncation chip), and localizes the period with `translatePeriod(proj.period.replace('Present', present), locale)`.
- Threaded `present` and `locale` props through `ProjectRow` ‚Üí `ProjectCard` (kept message-key surface minimal).
- Removed `MAX_PROJECTS`/`MAX_TECH_TAGS` constant declarations from cv-styles.ts (CvProjects was the only consumer).
- `CvDocument.tsx` now passes `locale` to `<CvProjects>`.

**Task 3 ‚Äî GitHub in header** ‚Äî commit `a8d257b`
- `CvHeader.tsx` imports `GITHUB_URL`/`GITHUB_DISPLAY` and renders a GitHub `<Link>` after LinkedIn, separated by a `|`, matching the existing separator/link pattern. `contactRow` already has `flexWrap: 'wrap'` so the extra item wraps cleanly.
