<objective>
Center the 7 inline navigation links in the desktop navbar row so the layout reads
balanced — links sit centered between the "JK" brand (left) and the theme/language
controls (right), eliminating the large dead gap that currently appears between the
last link ("Contact") and the controls.

The defect's root cause (verified): SGDS `<sgds-mainnav>` renders its shadow-DOM row as
`.navbar-brand` + `.navbar-body` > `.navbar-nav` (a flex row) containing the default
`<slot>` (our `InlineNav` wrapper) followed by `<slot name="end" class="slot-end">`
(the controls). `.slot-end` uses `margin-left: auto`, so it is pushed to the far right;
the default slot sizes to its content and stays left-aligned right after the brand. The
links therefore hug the brand and a dead gap opens before the controls.

The fix composes ENTIRELY within the `InlineNav` wrapper in `Navbar.tsx` (lines ~68-77):
make that slotted wrapper grow to fill the available middle space and center its links
inside, while keeping `flex-nowrap` + `overflow-x-auto` so the centered cluster still
falls back to horizontal scroll (NAV-05) when the 7 links cannot all fit.

Purpose: NAVBAL-01..04 — a balanced, professional desktop navbar with zero regression to
the v1.5 responsive behavior (phone drawer, NAV-05 overflow scroll, 44px taps, no page
x-overflow 320-1024px).
Output: Modified `InlineNav` wrapper in `Navbar.tsx` + a regression-guarding test in
`Navbar.test.tsx`. No new dependencies. No globals.css change expected (see Task 1 note).
</objective>
