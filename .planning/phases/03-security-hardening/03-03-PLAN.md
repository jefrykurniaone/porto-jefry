<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| middleware -> browser | Middleware emits the Content-Security-Policy header; the browser enforces which scripts/styles may execute. The inline theme-init script must be explicitly authorized. |
| inline script -> DOM | The theme-init `<script>` (layout.tsx:68) reads localStorage and toggles a class before hydration; if blocked it silently fails (theme flash). |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-3-02 | Tampering | prod script-src in buildCsp (middleware.ts:28) | mitigate | Authorize the theme-init inline script with a static SHA-256 hash matching the live script body, kept alongside `'strict-dynamic'`; the hash is load-bearing because x-nonce is null on Vercel at runtime (D-09). Hash re-verified programmatically against deployed HTML (Task 3, D-10). |
| T-3-04 | Tampering | style-src in buildCsp (middleware.ts:17,29) | accept | `'unsafe-inline'` retained for styles (SGDS shadow-DOM + next/image inline styles need it). CSS cannot execute JS, so the XSS risk is low; no nonce is added to style-src to avoid silently disabling unsafe-inline (D-07). |
| T-3-05 | Information Disclosure | Removal of unused fonts.googleapis.com / fonts.gstatic.com | mitigate | Removing unused remote font origins shrinks the CSP allowlist (least privilege); confirmed safe by font audit CF-2 (no @font-face/gstatic/googleapis in SGDS CSS), so no font rendering regression (D-05, D-06). |
</threat_model>
