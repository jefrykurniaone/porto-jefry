## Design System

| Property | Value |
|----------|-------|
| Tool | SGDS web components and SGDS Tailwind v4 utilities. shadcn is not applicable because Phase 5 is a locked SGDS migration. |

**SGDS setup contract:**

- Process `css/utility.css` through Tailwind v4 and `@tailwindcss/postcss`; never import it from JavaScript.
- Register SGDS custom elements once through a client-only loader, for example `src/app/sgds.tsx`, using dynamic import of `@govtechsg/sgds-web-component`.
- Add `src/types/sgds.d.ts` with `import '@govtechsg/sgds-web-component/types/react';`.
- Add `suppressHydrationWarning` on direct `<sgds-*>` elements rendered by Next.js.
- Validate direct `<sgds-*>` usage in this React 18 app after the foundation task. If direct custom events or boolean attributes fail, introduce thin client wrappers that still render SGDS web components and isolate `addEventListener` wiring.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label micro gaps, badge icon spacing, timeline rail offsets |
| sm | 8px | Compact inline spacing, badge groups, nav item internal gaps |
| md | 16px | Default text gaps, button groups, card inner micro layout |
| lg | 24px | Component padding, grid gutters, section heading-to-content gaps |
| xl | 32px | Card grid gaps, grouped content blocks, section internal separation |
| 2xl | 48px | Section vertical padding on mobile/tablet, major content groups |
| 3xl | 64px | Desktop section breaks and hero breathing room |

Exceptions: minimum touch targets are 44px for icon-only controls such as theme toggle, mobile menu, and BackToTop. Hero may use `min-height: 100svh` or equivalent SGDS utility/arbitrary value to preserve above-the-fold composition.

**SGDS utility mapping:**

| Context | Required utility family | Default |
|---------|-------------------------|---------|
| Page container | `.sgds-container` | Use for every portfolio page and section wrapper. |
| Section padding | `sgds:py-layout-lg` desktop, `sgds:py-layout-md` mobile/tablet | No raw `py-20` after migration. |
| Section alternation gap | `sgds:gap-layout-md` | Use between major layout areas. |
| Card padding | `sgds:p-component-xs` | Use SGDS card internals where available instead of custom padding. |
| Card/grid gap | `sgds:gap-layout-sm` or `.sgds-grid` gutters | Prefer SGDS grid before raw `sgds:grid-cols-*`. |
| Text stack gap | `sgds:gap-text-md` | Use between heading, subtitle, and body copy. |
| Badge/chip gap | `sgds:gap-text-xs` or raw `sgds:gap-2` | Use only when semantic gap is too large. |

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Label / metadata | 14px | 400 regular or 600 semibold | 1.4 |
| Body | 16px | 400 regular | 1.5 |
| Heading | 24px | 600 semibold | 1.2 |
| Display | 48px | 600 semibold | 1.2 |

**Scale:** exactly 4 sizes: 14, 16, 24, 48px. Exactly 2 weights: 400 and 600. Do not introduce 700/bold in migrated UI.

**SGDS utility mapping:**

- Display hero name: `sgds:text-display-md sgds:font-semibold sgds:leading-2-xl sgds:tracking-tighter sgds:text-display-default`.
- Section headings: `sgds:text-heading-lg sgds:font-semibold sgds:leading-lg sgds:tracking-tight sgds:text-heading-default`.
- Card titles: `sgds:text-heading-sm` or `sgds:text-subtitle-md` with `sgds:font-semibold`.
- Body copy: `sgds:text-body-md sgds:leading-xs sgds:text-body-default`.
- Supporting copy and metadata: `sgds:text-label-sm` or `sgds:text-caption-md` with `sgds:text-muted`.
- Buttons and nav labels must use SGDS component typography. Do not override component internals with custom CSS.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | SGDS `default` background tokens: `sgds:bg-default`, `sgds:text-default` | Body background, primary content surfaces, main reading areas |
| Secondary (30%) | SGDS `alternate` and `surface` tokens: `sgds:bg-alternate`, `sgds:bg-surface-default`, `sgds:bg-surface-raised`, `sgds:border-default` | Alternating sections, cards, nav surface, footer surface, timeline panels |
| Accent (10%) | SGDS `primary` tokens with default SGDS palette. No custom brand override. | Primary CTA, active/current nav state, focus rings, timeline dots, section icon highlights, verified link emphasis |
| Destructive | SGDS `danger` tokens and `<sgds-alert variant="danger">` | CV generation failure and true error states only |

Accent reserved for: primary Hero CTA, active nav item, focus-visible rings, timeline nodes, selected/toggled theme state, section icon highlights, and standalone important links. Do not use accent for every card border, every badge, or decorative backgrounds.

**Colour rules:**

- Use theme-aware SGDS tokens only. Remove all `dark:*` classes from migrated UI.
- Use `primary` for brand emphasis and actions; use `accent` only for informational link emphasis where SGDS component semantics call for it.
- Use `neutral` or `outlined` badges for repeated technology tags to avoid visual noise.
- Use `danger` only for errors or destructive actions. This phase has no destructive user actions.
- Use `warning` only if a non-blocking caution state appears. The error boundary should use danger or neutral error treatment, not decorative yellow.
- Do not introduce custom hex colours or custom product primary overrides in this phase.

## Layout Contract

### App shell

Use a public-facing simple app layout:

1. Optional SGDS masthead: **omit by default**. This site is a personal portfolio on Vercel, not a `.gov.sg` agency site. Rendering `<sgds-masthead>` would imply official government ownership. Add it only if the deployment context changes to an official agency site.
2. Navigation: use `<sgds-mainnav>` for the desktop shell with brand slot `JK.` and in-page nav items.
3. Mobile navigation: keep a custom React hamburger menu as the locked CONTEXT.md exception. Style it with SGDS utilities and SGDS icons. It must preserve focus trap, Escape close, focus return, and hash scrolling.
4. Main content: each section uses `.sgds-container`; do not recreate max widths with raw Tailwind utilities.
5. Footer: use a custom `<footer>` styled with SGDS utilities unless `<sgds-footer>` can be configured with truthful, valid links. Do not ship `<sgds-footer>` with placeholder `#` privacy or terms links, and do not imply `Government of Singapore` ownership. This is the only footer exception allowed for content truthfulness.

### Section rhythm

- Alternate page sections between `sgds:bg-default` and `sgds:bg-alternate` to preserve the current visual rhythm.
- Section order remains: Hero, About, Experience, Education, Skills, Projects, Certifications, Contact.
- Hero remains centred and above the fold. Sections after Hero are document-flow sections with anchored IDs.
- Not-found and error pages are in scope for SGDS polish because they are user-visible UI surfaces.

### Grid rules

| Content area | Mobile | Tablet | Desktop |
|--------------|--------|--------|---------|
| Hero content | 4-column centred stack | 8-column centred stack | centred 8 of 12 columns |
| About | single card stack | 8-column centred | 8 of 12 columns centred |
| Experience timeline | single timeline column | single timeline column | 8 of 12 columns centred |
| Education | 1 column | 2 columns where content permits | 2 columns within 8 of 12 columns |
| Skills | 1 column | 2 columns | 3 columns |
| Projects | 1 column | 2 columns | 3 columns |
| Certifications | 1 column | 1 to 2 columns | centred narrow column unless more certifications are added |
| Contact | 1 column | 2 columns | 4 equal columns |

Use `.sgds-grid` and `sgds-col-*` classes first. Only use `sgds:grid-cols-*` for auto-fit card rows that SGDS grid cannot express.

## Interaction Contract

### Navigation

- Anchor links use plain `#id` hashes and smooth scroll. Preserve Phase 1 hash consistency.
- Navbar active state may follow the current hash or scroll position, but no active state is preferable to an incorrect one.
- Sticky nav must not cover anchored section headings. Add scroll margin to sections with SGDS utilities or global base CSS.
- Mobile menu opens from the nav toggle, traps focus inside the menu, closes on Escape, closes after selecting a section, and returns focus to the toggle.

### CV download

- Primary action label: `hero.cta_cv` (`Download CV` / `Unduh CV`).
- While fetching: use `<sgds-button loading>` where React 18 support validates; otherwise a wrapper that sets the SGDS `loading` property by ref. Loading text remains available to assistive tech via `hero.downloading_cv`.
- Prevent duplicate requests while loading.
- On failure: show `<sgds-alert show variant="danger">` directly under the Hero CTA group. Preserve `role="alert"` and `aria-live="polite"` in the slotted body.
- Auto-dismiss is allowed after 5 seconds only for the CV inline error because Phase 2 already established this behaviour. Do not auto-dismiss critical page errors.

### Theme toggle

- Storage key: `sgds-theme`.
- Valid values: `day` and `night` only.
- Runtime owner: `document.documentElement.classList.toggle('sgds-night-theme', theme === 'night')`.
- Apply a nonce-safe pre-hydration class initialiser before first paint to avoid a saved night-mode flash. Use the existing CSP nonce path from middleware and layout.
- Toggle label updates before or at the same time as the visual state. Icon-only control must expose a translated `ariaLabel`.

### Language toggle

- Preserve existing `next-intl` routing and current path when switching EN/ID.
- Do not hardcode labels beyond the visible locale abbreviations `EN` and `ID`.
- Keep translated `aria-label` via `lang.switch`.

### Cards and links

- Use clickable card surfaces only when the card has exactly one destination. Otherwise keep links/buttons explicit.
- External links open in a new tab with `rel="noopener noreferrer"`.
- Technology badges are non-interactive. Do not make tags clickable in this phase.

## Responsive Behavior

| Breakpoint | Contract |
|------------|----------|
| <512px | One-column content; CTA buttons stack full width or wrap with 44px minimum height; nav uses custom hamburger; no horizontal scrolling. |
| 512px to 767px | Two-column layout is allowed only for small repeatable cards if labels remain readable. |
| 768px to 1023px | Content can use 8-column SGDS grid; card sections may use two columns; nav may remain hamburger to avoid seven-link overflow. |
| >=1024px | Desktop nav visible; sections use 12-column SGDS grid with centred 8-column reading areas and 3 or 4 column card grids. |
| >=1440px | `.sgds-container` maxes at SGDS 2XL width. Do not stretch text beyond readable line length. |

Use SGDS responsive utility syntax as `sgds:md:flex-row`, never `md:sgds:flex-row`.

## Accessibility Requirements

- Keep the skip link as the first focusable control inside `<body>` and restyle it with SGDS utility classes.
- Every section keeps a semantic `<section id="...">` and a visible heading.
- Heading order remains logical: Hero H1, section H2, cards H3/H4 as needed.
- Preserve `alt={t('photo_alt')}` for the profile image.
- All icon-only controls use `<sgds-icon-button>` with `ariaLabel` or an equivalent accessible custom button.
- Touch targets are at least 44px by 44px for mobile controls.
- Focus states must be visible and use SGDS primary focus treatment. Do not remove outlines without replacement.
- Do not rely on colour alone. Error, empty, and loading states need text.
- All user-facing copy must come from `useTranslations()` or `getTranslations()`.
- Test keyboard paths: skip link, desktop nav, mobile menu open/close, theme toggle, language toggle, CV button, BackToTop, contact links, not-found return, and error reset.
- Maintain WCAG AA contrast, including in night mode and on alternate sections.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | `hero.cta_cv`: `Download CV` / `Unduh CV` |
| Empty state heading | `No {section} to show` / `Belum ada {section} yang ditampilkan` if any static collection unexpectedly has no items |
| Empty state body | `Check back later for updates.` / `Kunjungi lagi nanti untuk pembaruan.` |

| Destructive confirmation | Not applicable. Phase 5 introduces no destructive user actions. |

**Tone:** clear, direct, bilingual, and professional. Use sentence case for any new or changed labels. Do not conduct a broad portfolio writing audit in this phase because UK English writing standards were explicitly deferred.

**Existing key treatment:**

- Preserve existing translation keys where possible to minimise scope.
- If button labels are touched during migration, normalise only labels and controls to sentence case: `View my work`, `Download CV`, `Contact me`, `Try again`, `Return home`.
- Do not hardcode component text such as alert titles, button labels, aria labels, or empty states.
- Update `footer.built_with` if the visible footer keeps this line: `Built with Next.js and SGDS` / `Dibangun dengan Next.js dan SGDS`.

## Theming and Dark Mode

| Requirement | Contract |
|-------------|----------|
| Theme mechanism | `.sgds-night-theme` on `<html>` only. No `next-themes`, no Tailwind `dark:` utilities. |
| Persistence | `localStorage['sgds-theme']` with values `day` or `night`. Invalid values fall back to `day`. |
| Initial paint | Existing CSP nonce is used for a pre-hydration script that applies `.sgds-night-theme` before React hydrates. |
| Toggle UI | `<sgds-icon-button>` with sun/moon icons, translated labels, and 44px touch target. |
| Token source | SGDS day and night theme CSS. Do not override primary colour scale. |
| Components | All migrated surfaces use SGDS semantic tokens so day/night changes require no per-component class changes. |

## Validation Contract

Before marking Phase 5 visually complete, verify:

1. `/en` and `/id` render with no hydration errors from SGDS custom elements.
2. Day and night modes render with no flash beyond first paint and persist after reload.
3. Browser console shows no CSP violations for SGDS loader, theme script, Analytics, or styles.
4. No migrated UI file contains `dark:` utilities, `useTheme`, `ThemeProvider`, or `next-themes` imports.
5. All migrated UI components use SGDS components where an equivalent exists, with documented exceptions only for timeline rail, BackToTop logic, mobile hamburger, GitHub icon, masthead omission, and truthful footer fallback.
6. Lint, typecheck, tests, coverage, and build pass in project order: `npm run lint`, `npx tsc --noEmit`, `npm run test:coverage`, `npm run build`.
7. Keyboard checks pass for desktop and mobile navigation, theme/language toggles, CV download, contact cards, 404 page, and error page.
